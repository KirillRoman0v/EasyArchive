Архиватор, здесь.
Реализовывать будем так:
1. Сначала подсчитывается частота каждого встречающегося в файле символа(каждый символ это 1 байт)

2. Исходя из частоты, каждый символ получает новое значение, высчитываемое как менее местоемкое.
   Эти значения должны быь переменной величины, то есть более частые символы будут кодироваться меньшим количеством битов чем менее частые.
   Таким образом размер файла будет уменьшен.

3. В начале архива будет алфавит, состоящий из информации о символах, встречающихся в файле и их коде в архиве.
   Соответственно деархиватор, принимая на вход архив, будет сначала инициализировать алфавит, потом с его помощью
   обрабатывать данные, идущие в архиве следом, постепенно восстанавливая исходный файл из архива.
